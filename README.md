# Гайд по переезду в Словакию

Открытый, поддерживаемый сообществом гайд для тех, кто переезжает в Словакию: ВНЖ, банки, водительские права, налоги, транспорт, учёба, работа и бытовые мелочи.

Сайт: **https://slovakiaguide.github.io**

[![PRs Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg)](#как-внести-вклад)
[![Made with Jekyll](https://img.shields.io/badge/made%20with-Jekyll-red)](https://jekyllrb.com/)

---

## Зачем этот гайд

Информация о переезде разбросана по чатам, форумам и устаревшим статьям. Здесь мы собираем её в одном месте, проверяем на собственном опыте и держим актуальной. Каждая страница — это чей-то живой опыт прохождения процедуры: какие документы реально спросили, в каком окне, сколько ждали, что пошло не так.

## Кому это пригодится

- Тем, кто только планирует переезд и не знает, с чего начать.
- Тем, кто уже в Словакии и столкнулся с конкретной процедурой (продление ВНЖ, обмен прав, регистрация на slovensko.sk).
- Тем, кто хочет помочь другим — поделиться свежим опытом или поправить устаревшее.

## Как внести вклад

Гайд живёт за счёт сообщества. **Любая правка ценна** — от опечатки до новой главы.

### Что можно сделать

- ✏️ **Поправить опечатку или устаревшую информацию** — открой страницу на сайте, нажми «Edit on GitHub» (или найди файл в [`_pages/`](_pages/)) и предложи правку.
- ➕ **Добавить свой опыт** к существующей странице: «У меня в феврале 2026 в OCP в Братиславе спросили ещё вот это…».
- 📄 **Написать новую страницу** про процедуру, которой ещё нет в гайде.
- 🐛 **Сообщить о проблеме** — [создай issue](../../issues/new), если не готов править сам: опиши, что устарело или непонятно.
- 💬 **Задать вопрос** в [Discussions](../../discussions), если не уверен, куда это положить.

### Как именно править

Подробная инструкция — в [CONTRIBUTING.md](CONTRIBUTING.md). Если коротко:

1. Форкни репозиторий и создай ветку.
2. Поправь нужный файл в `_pages/` (это обычный HTML с YAML-шапкой).
3. Открой Pull Request с коротким описанием: **что меняется и откуда инфа** (личный опыт / ссылка на закон / скрин из ÚHCP).
4. Мы обсудим в PR и смержим.

Не страшно, если что-то сделано «не так» — поможем причесать в ревью. Главное — содержание.

### Принципы

- **Свежесть важнее красоты.** Лучше короткая заметка «в марте 2026 на Hraničná работало вот так» сегодня, чем идеальная статья через полгода.
- **Указывай источник.** Личный опыт (с датой и местом), официальный сайт, конкретный § закона — всё подходит, но читателю нужно понимать, откуда инфа.
- **Не давай юридических советов.** Мы делимся опытом, а не консультируем. Спорные моменты — со ссылкой на первоисточник.
- **Уважай людей в примерах.** Никаких чужих имён, номеров документов, фото без согласия.

## Структура репозитория

```
_pages/        — содержимое гайда (одна страница = один HTML-файл)
_includes/     — шапка, навигация, футер
_layouts/      — шаблоны
assets/        — стили, JS, картинки
_config.yml    — настройки Jekyll
```

## Локальный запуск

Нужен, только если хочешь увидеть правки до PR. Для опечаток достаточно редактора прямо на GitHub.

```bash
bundle install
bundle exec jekyll serve
# открой http://localhost:4000
```

Подробнее — в [CONTRIBUTING.md](CONTRIBUTING.md#локальная-сборка).
