# Как внести вклад

Спасибо, что хочешь помочь гайду! Этот файл — практическая инструкция: как править, как оформлять, как открывать PR.

Если что-то непонятно — открой [Discussion](../../discussions) или просто отправь PR как есть, поможем причесать.

## Способы вклада

### 1. Быстрая правка через GitHub (для опечаток и мелочей)

1. Открой нужную страницу на сайте.
2. Найди тот же файл в [`_pages/`](_pages/) (имя файла = последняя часть URL).
3. Нажми карандаш ✏️ в правом верхнем углу — GitHub предложит сделать форк и открыть редактор.
4. Поправь, внизу нажми **Propose changes** → **Create pull request**.

### 2. Полноценная правка локально (для новых страниц или большого объёма)

```bash
git clone git@github.com:slovakiaguide/slovakiaguide.github.io.git
cd slovakiaguide.github.io
git checkout -b my-edit
# правишь файлы
bundle exec jekyll serve  # проверяешь на http://localhost:4000
git commit -am "Add section about X"
git push origin my-edit
# открываешь PR на GitHub
```

### 3. Issue — если не готов править сам

[Открой issue](../../issues/new) и опиши: какая страница, что не так, что знаешь по делу. Кто-нибудь подхватит.

## Что писать в PR

Шаблон описания (его подставит [`.github/pull_request_template.md`](.github/pull_request_template.md)):

```
## Что меняется
Короткое описание правки.

## Источник
- личный опыт: где, когда (например: «OCP Petržalka, март 2026»)
- ссылка на официальный документ / § закона / скриншот формы
- ссылка на тред в чате/форуме (если применимо)

## Чек-лист
- [ ] Указал дату актуальности (если процедура меняется)
- [ ] Проверил локально (если правил вёрстку)
```

## Стиль контента

### Тон

- Пишем как другу, который переезжает: коротко, по делу, без воды и канцелярита.
- «Ты», а не «Вы».
- Английские/словацкие термины оставляем в скобках при первом упоминании: `подтверждение об отсутствии задолженности (potvrdenie o nedoplatkoch)`.

### Актуальность

- Указывай дату на странице или у конкретного шага: `_Актуально на март 2026._`
- Если переписываешь раздел — обнови дату.
- Старые скриншоты лучше, чем никаких; новые — лучше старых.

### Источники и ссылки

- Официальные документы — со ссылкой на конкретную страницу `slovensko.sk`, `minv.sk`, `financnasprava.sk`.
- Цены, сроки и адреса часто меняются — пиши в формате «было X на момент Y».
- Не вставляй ссылки на платные консультации и реферальные ссылки.

### Приватность

- Никаких чужих имён, родинных чисел, номеров документов, домашних адресов.
- Скриншоты — с замазанными личными данными.
- Свои примеры — обезличивай: «Иван Иванов, 1.1.1990» → «Имя Фамилия, дата рождения».

## Структура страницы

Каждая страница — HTML-файл в `_pages/` с YAML-шапкой:

```yaml
---
layout: page
title: Название страницы            # обязательно, идёт в <h1>
permalink: /url-страницы/           # обязательно, со слешем на конце
description: Краткое описание       # опционально, для SEO и сниппетов
order: 10                            # опционально, порядок в навигации
toc: true                            # опционально, оглавление справа
nav: false                           # опционально, скрыть из бокового меню
---
```

### Заголовки

- `<h1>` рендерится автоматически из `title`. **Не пиши ещё один `h1` в теле.**
- В тексте начинай с `<h2>`, дальше `<h3>`, `<h4>` — для оглавления нужны атрибуты `id`:

```html
<h2 id="documents">Документы</h2>
<p>…</p>

<h3 id="passport">Паспорт</h3>
<p>…</p>
```

Подробнее: [`.claude/rules/design-system.md`](.claude/rules/design-system.md).

### Навигация

Группы и порядок пунктов в левом меню — в [`_includes/left-nav.html`](_includes/left-nav.html), формат:
`"Группа|slug1,slug2;Группа2|slug3"`. `slug` — имя файла в `_pages/` без `.html`.

## Локальная сборка

```bash
bundle install                # один раз
bundle exec jekyll serve      # сервер на localhost:4000 с автоперезагрузкой
bundle exec jekyll build      # просто собрать в _site/
```

Если Ruby не установлен — проще всего через [rbenv](https://github.com/rbenv/rbenv) или `brew install ruby`.

## Кодекс поведения

Будь доброжелателен. Гайд читают люди в стрессовой ситуации переезда — давай не добавлять им токсичности. Грубость, дискриминация и реклама — повод закрыть PR без обсуждения.
